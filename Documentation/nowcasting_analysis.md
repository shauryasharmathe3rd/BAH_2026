# Solar Flare Nowcasting Performance Analysis

This report documents the solar flare nowcasting logic implemented in the codebase, the purpose and content of [verify_combined_instruments.py](../verify_combined_instruments.py), and the verification results of adding the nowcasting evaluation logic block to the Jupyter Notebook [analysis.ipynb](../analysis.ipynb).

---

## 1. What is in `verify_combined_instruments.py`?

[verify_combined_instruments.py](../verify_combined_instruments.py) is a standalone verification script designed to validate joint soft and hard X-ray nowcasting across multiple instrumentation datasets (SoLEXS and HEL1OS). Key details:
*   **Data Processing**: It defines helper functions to load FITS data for both SoLEXS (thermal soft X-ray) and HEL1OS (non-thermal hard X-ray).
*   **Algorithmic Pipelines**:
    *   Runs the **Adaptive Rolling Sigma-Threshold Nowcaster** (Algorithm 1) on SoLEXS data.
    *   Smooths the spiky hard X-ray HEL1OS counts and runs the same **Rolling Sigma Nowcaster** on it.
*   **Catalogue Merging**: Combines both independent event catalogues into a single **Master Catalogue** using the physical constraints of the **Neupert Effect** (HXR peak precedes the SXR peak during the flare rise phase).
*   **Output**: Saves the matched events (identifying `joint` events, `soft_only` events, and `hard_only` events) to `master_flare_catalogue.csv`.

## 2. Nowcasting Logic Summary

Unlike forecasting models (e.g. LSTMs) that predict future time steps, **nowcasting** focuses on real-time statistical detection of events currently in progress. The following methods are implemented in [nowcast.py](../nowcast.py):

1.  **Algorithm 1: Adaptive Rolling Sigma-Threshold**
    *   Maintains a rolling window of background counts (default size $600$ seconds).
    *   Triggers a flare when count rates exceed $\mu + k \cdot \sigma$ (default $k=3.5$) for a sustained duration ($\ge 5$ seconds).
    *   Flags event end when counts drop back below $\mu + 1.5 \cdot \sigma$.
2.  **Algorithm 2: Smoothed First-Derivative Thresholding**
    *   Computes numerical first-derivative $\frac{dC}{dt}$ on smoothed counts.
    *   Triggers when rise rate exceeds a steep threshold (default $10.0\text{ counts/s}^2$) for $\ge 3$ seconds.
3.  **Real-Time Probability Nowcaster (`Nowcaster` Class)**
    *   Computes a statistical Z-score for the latest count rate relative to dynamic background stats.
    *   Uses a sigmoid function centered at the multiplier $k$ to output a probability in the range $[0.0, 1.0]$.

## 3. Verification Cell Added to `analysis.ipynb`

To verify the nowcasting performance directly in the project notebook, a new code block (Cell 24) has been added to [analysis.ipynb](../analysis.ipynb) with the following structure:
*   Loads SoLEXS fits table data.
*   Runs Algorithm 1 (Adaptive Rolling Sigma-Threshold) and Algorithm 2 (Smoothed First-Derivative) on the dataset.
*   Compares quiet and flare onset windows using the `Nowcaster.predict` method.
*   Plots both algorithm triggers and thresholds graphically using `matplotlib`.

> [!NOTE]
> During verification, missing module imports (`architecture`, `models`, and `sliding_window`) were identified because their source `.py` files had been cleaned or deleted. To resolve this, their compiled `.pyc` files were extracted from `__pycache__` and placed directly into the project root as `architecture.pyc`, `models.pyc`, and `sliding_window.pyc`. Python can now import them successfully, restoring complete execution capacity for both the notebook and standalone scripts.

## 4. Verification Run Results

The execution of the nowcasting verification block on SoLEXS SDD2 data yields the following outputs:

### A. Numerical Values

*   **Dataset Baseline**:
    *   Mean background counts: $516.18\text{ cts/s}$
    *   Standard Deviation: $614.13\text{ cts/s}$
*   **Real-time Probability Nowcaster**:
    *   **Quiet Window** (counts range $459.0$ - $543.0\text{ cts/s}$): **$1.4259\%$** flare probability.
    *   **Flare Onset Window** (counts range $2036.0$ - $2420.0\text{ cts/s}$): **$16.1059\%$** flare probability (onset trigger).
*   **Algorithmic Catalogues**:
    *   **Rolling Sigma Algorithm**: Detected **$11$** distinct flare events.
    *   **Smoothed Derivative Algorithm**: Detected **$50$** fine-grained rise events (earlier trigger latency).
